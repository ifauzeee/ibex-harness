## Why This Milestone Exists

Without CI gates, broken internal links and accessibility regressions reach preview and production silently. Go CI already blocks bad merges — docs need equivalent `docs-checks.yml`.

lychee on built HTML catches stub pages linking to removed paths. Lighthouse enforces a11y = 100 per Matte Graphite contract.

Depends on D.4.1 preview URL pattern for Lighthouse job target.

---

## Non-Goals

- E2E Playwright in this milestone (D.5.1 adds visual tests)
- Deploying to production from CI

## Testing Requirements

- **`test_lychee_blocks_broken_links`**: PR with broken href fails lychee job
- **`test_lighthouse_blocks_a11y_regression`**: PR removing alt text fails Lighthouse
- **`verifyDocsChecks_PathFilter`**: Go-only PR does not trigger docs-checks

---

## Acceptance Criteria

- [ ] PR with intentionally broken link is blocked by lychee
- [ ] PR with a11y regression is blocked by Lighthouse
- [ ] Thresholds documented: perf ≥ 90, a11y = 100, best-practices ≥ 95, SEO = 100
