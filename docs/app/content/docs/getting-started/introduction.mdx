IBEX Harness is a self-hosted AI agent platform: an authenticated LLM proxy, multi-tenant auth, and (in later phases) persistent agent memory.

<Callout type="note">
  Phase 1 ships the **proxy** and **auth** services. Chat completion routing returns `501 PROVIDER_NOT_CONFIGURED` until Phase 2 wires a provider adapter.
</Callout>

## What you can do today

- Validate bearer tokens and agent identity on every proxied request
- Run the full stack locally with Docker Compose
- Enforce per-organization rate limits (Redis-backed, fail-open)
- Browse this docs site with search, OG previews, and Matte Graphite UI

## Architecture at a glance

- **Client** sends authenticated requests to the **proxy** (port 8080)
- **Proxy** validates tokens via **auth** (gRPC 9091) and rate limits via **Redis**
- **Auth** reads org/agent/token data from **Postgres**

## Verify the proxy is up

After `make compose-dev-up`, the proxy health endpoint should respond:

```bash
curl -s http://localhost:8080/health
```

Expected: HTTP 200 with a JSON body reporting service health.

## Rate limiting

The proxy applies per-tenant rate limits on incoming requests. Limits are configurable via environment variables and enforced before request routing — see [Rate limiting](/docs/proxy/rate-limiting) for the full reference.
