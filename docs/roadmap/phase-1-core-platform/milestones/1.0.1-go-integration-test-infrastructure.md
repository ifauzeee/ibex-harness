# Milestone 1.0.1: Go Integration Test Infrastructure

**Status:** Planned
**Goal:** [1.1 Persistence and auth data plane](../goals.md)
**Phase:** 1 — Core Platform

## Why This Milestone Exists

Every subsequent milestone (1.1.3, 1.2.1, 1.2.2, 1.3.1) requires integration tests
against real Postgres and Redis. Without a shared, documented infrastructure for
running those tests, each milestone will invent its own approach, creating drift.

`TESTING_STRATEGY.md §4` mandates `testcontainers-go` for Postgres and Redis.
`AGENTS.md §9` forbids mocking RLS behavior. Neither document tells you *how* to
wire testcontainers into the Go test suite, how to tag integration tests, or how
to run them in CI. This milestone resolves all three.

### 1. Resolve the integration test tagging decision (pending in `decisions.md`)

Choose and document one approach:

**Option A (recommended):** `//go:build integration` build tag on test files.

```go
//go:build integration

package repository_test
```

Run with: `go test -tags=integration ./...`

**Rationale:** Keeps integration tests in the same package as the code they test.
Standard Go idiom. CI can run `go test ./...` (unit only) and
`go test -tags=integration ./...` (all) as separate steps.

Document the decision in `phase-1-core-platform/decisions.md`.

### 3. Create `internal/testutil/` shared package

Create `internal/testutil/postgres.go`:

```go
// Package testutil provides shared test helpers for integration tests.
// All functions in this package require the `integration` build tag.
package testutil

// StartPostgres starts a Postgres container with pgvector extension enabled,
// runs migrations, and returns a connection string and cleanup function.
// The container is started with the same image as infra/compose/dev.
func StartPostgres(t testing.TB, migrationsDir string) (dsn string, cleanup func())

// StartRedis starts a Redis container and returns a connection URL and cleanup.
func StartRedis(t testing.TB) (url string, cleanup func())

// MustSetOrgContext sets SET LOCAL app.current_org_id on the given transaction.
// Fails the test immediately if the SET fails.
func MustSetOrgContext(t testing.TB, tx pgx.Tx, orgID uuid.UUID)
```

Create `internal/testutil/fixtures.go`:

```go
// SeedOrganization inserts a test organization and returns its ID.
func SeedOrganization(t testing.TB, db *pgxpool.Pool) uuid.UUID

// SeedToken inserts a hashed token for the given org and returns
// the plaintext token value (for use in test requests) and the token ID.
func SeedToken(t testing.TB, db *pgxpool.Pool, orgID uuid.UUID,
    permissions int64) (plaintext string, tokenID uuid.UUID)
```

### 4. Create CI job for integration tests

Add to `.github/workflows/ci.yml`:

```yaml
go-integration:
  name: go-integration
  runs-on: ubuntu-latest
  continue-on-error: true  # advisory until Phase 1 complete
  services:
    postgres:
      image: pgvector/pgvector:pg16
      env:
        POSTGRES_USER: ibex
        POSTGRES_PASSWORD: ibex
        POSTGRES_DB: ibex_test
      ports: ["5432:5432"]
      options: >-
        --health-cmd pg_isready
        --health-interval 10s
        --health-timeout 5s
        --health-retries 5
    redis:
      image: redis:7-alpine
      ports: ["6379:6379"]
  steps:
    - uses: actions/checkout@v6
    - uses: actions/setup-go@v5
      with:
        go-version-file: go.mod
    - name: Run migrations
      run: make db-migrate
      env:
        POSTGRES_DSN: postgres://ibex:ibex@localhost:5432/ibex_test
    - name: Integration tests
      run: go test -tags=integration -race -timeout=120s ./...
      env:
        POSTGRES_DSN: postgres://ibex:ibex@localhost:5432/ibex_test
        REDIS_URL: redis://localhost:6379/0
```

### 5. Write a smoke integration test to validate the infrastructure

`services/auth/internal/testutil/infra_test.go`:

```go
//go:build integration

func TestIntegrationInfraSmoke(t *testing.T) {
    dsn, cleanup := testutil.StartPostgres(t, "../../../../infra/migrations/postgres")
    defer cleanup()

    pool, err := pgxpool.New(context.Background(), dsn)
    require.NoError(t, err)
    defer pool.Close()

    // Verify RLS: org A cannot see org B rows
    orgA := testutil.SeedOrganization(t, pool)
    orgB := testutil.SeedOrganization(t, pool)

    // Insert row for org A
    _, err = pool.Exec(ctx, `
        INSERT INTO ibex_core.organizations (id, name, slug, tier)
        VALUES ($1, 'Org A', 'org-a', 'free')
    `, orgA)
    // ... verify org B context cannot read org A row
}
```
