# Milestone 1.2.2: Proxy Request Normalization

**Status:** Planned  
**Goal:** [1.2 Proxy platform integration](../goals.md#goal-12-proxy-platform-integration)

## Summary

Parse OpenAI-compatible chat completion request bodies into an internal normalized type. Return 400 on malformed JSON/shape. **Do not** call any LLM provider (Phase 2).

## Branch

`feature/m1-2-2-proxy-request-normalization`

3 days

## Tasks

- [ ] Handler `POST /v1/chat/completions` (or documented path) behind auth middleware
- [ ] Validate Content-Type, body size limits per SECURITY.md
- [ ] Return 501 or documented stub response indicating provider not configured — **not** fake success
- [ ] Unit tests: valid body, invalid JSON, missing model, oversized body
