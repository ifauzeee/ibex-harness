# Milestone 1.2.2: Proxy Request Normalization

**Status:** Complete  
**Goal:** [1.2 Proxy platform integration](../goals.md#goal-12-proxy-platform-integration)  
**Phase:** 1 — Core Platform

## Summary

Parse OpenAI-compatible `POST /v1/chat/completions` bodies into `internal/llm` types after auth. Malformed JSON → **400** `INVALID_JSON`. Valid parse → metadata-only log → **501** stub. No provider HTTP. Semantic validation and body limits → [1.2.3](1.2.3-proxy-input-validation-and-stable-error-envelope.md).

## Definition of done

- [x] Valid authenticated JSON → parsed + **501** `PROVIDER_NOT_CONFIGURED`
- [x] Malformed JSON → **400** `INVALID_JSON` with stable envelope
- [x] Logs contain model/message_count/stream/org_id only — never content
- [x] No outbound provider HTTP
- [x] `proxy-auth-smoke` / `go-services (proxy)` green

## Deferred (1.2.3)

- Body size limit, Content-Type 415, 413
- Required `model`, role enum, message bounds, `field_errors`
- `X-IBEX-Agent-ID` enforcement
- Response header middleware

## Risks

| Risk | Mitigation |
| --- | --- |
| Unbounded body until 1.2.3 | Document in ADR-0012; ship 1.2.3 next |
| Logging message content | Code review + structured log fields only |
| Path `/v1` vs `/proxy/v1` | ADR-0012 + API doc gateway alias |
