# Milestone 1.1.6: Argon2id Parameters and Crypto Policy ADR

**Status:** Planned
**Goal:** [1.1 Persistence and auth data plane](../goals.md)
**Phase:** 1 — Core Platform

## Why This Milestone Exists

`SECURITY.md §4.2` and `AGENTS.md §5.4` both mandate Argon2id for password
hashing. `DATABASE_SCHEMA.md` shows `password_hash TEXT` and `hash TEXT UNIQUE`
(for tokens). But nowhere in the codebase are the Argon2id parameters defined.

This is a security-critical gap. Argon2id parameters determine:
- **Security**: too-low parameters → brute-force feasible
- **Performance**: too-high parameters → DoS via login endpoint
- **Compatibility**: parameters must be stored with the hash for future upgrades

Without an ADR, every developer who implements a hash will choose different
parameters. A future parameter upgrade will be impossible without knowing what
parameters were used originally.

This milestone also covers the broader crypto policy: which primitives are
approved, which are forbidden, and how to use them correctly.

## Tasks

### 1. Write ADR-0008: Cryptography Policy

**Argon2id parameters (recommended baseline):**

```text
Memory:      64 MiB (65536 KiB)
Iterations:  3
Parallelism: 4
Salt:        16 bytes (crypto/rand)
Key length:  32 bytes
```

**Rationale:**
- OWASP recommends minimum 19 MiB; 64 MiB provides comfortable margin
- 3 iterations at 64 MiB takes ~100ms on a modern server (acceptable for login)
- Parallelism=4 matches typical server core count
- Parameters stored in PHC string format: `$argon2id$v=19$m=65536,t=3,p=4$...`

**PHC string format** (enables future parameter upgrades):

```text
$argon2id$v=19$m=65536,t=3,p=4$<base64_salt>$<base64_hash>
```

When parameters change, old hashes remain verifiable (parameters embedded in hash).
New hashes use new parameters. Rehash on next successful login.

**Other approved primitives:**
- JWT signing: RS256 with 2048-bit RSA keys (minimum); key rotation via JWKS
- Symmetric encryption: AES-256-GCM with random 96-bit nonce
- Hashing (non-password): SHA-256 (for content hashes, token prefixes)
- HMAC: HMAC-SHA256 (for webhook signatures)
- Random: `crypto/rand` only — never `math/rand`
- Constant-time comparison: `crypto/subtle.ConstantTimeCompare`

**Forbidden:**
- MD5, SHA-1 for any security purpose
- bcrypt (use Argon2id instead)
- PBKDF2 (use Argon2id instead)
- ECB mode for any cipher
- Custom crypto implementations

### 2. Create `internal/crypto/` package

```go
// Package crypto provides approved cryptographic primitives for IBEX Harness.
// All functions in this package implement the policy defined in ADR-0008.
// Do not use crypto primitives outside this package without an ADR.
package crypto

// HashPassword hashes a password using Argon2id with parameters from ADR-0008.
// Returns a PHC-format string that includes the parameters and salt.
// The returned string is safe to store in the database.
func HashPassword(password string) (string, error)

// VerifyPassword verifies a password against a PHC-format Argon2id hash.
// Uses constant-time comparison. Returns false (not error) for wrong password.
// Returns error only for malformed hash strings.
func VerifyPassword(password, phcHash string) (bool, error)

// HashToken hashes a token value using Argon2id.
// Token hashing uses the same parameters as password hashing.
func HashToken(plaintext string) (string, error)

// VerifyToken verifies a token against its Argon2id hash.
func VerifyToken(plaintext, phcHash string) (bool, error)

// GenerateRandomBytes generates n cryptographically secure random bytes.
// Panics if the system entropy source is unavailable (should never happen).
func GenerateRandomBytes(n int) []byte

// GenerateRandomBase62 generates a base62-encoded random string of the given byte length.
// The output length will be ceil(n * 4/3) characters.
func GenerateRandomBase62(byteLength int) string

// ConstantTimeEqual compares two strings in constant time.
// Use this instead of == for any security-sensitive comparison.
func ConstantTimeEqual(a, b string) bool
```

### 3. Write test vectors

```go
func TestArgon2idParameters(t *testing.T) {
    // Verify parameters match ADR-0008
    hash, err := HashPassword("test-password")
    require.NoError(t, err)

    // Verify PHC format
    assert.True(t, strings.HasPrefix(hash, "$argon2id$v=19$m=65536,t=3,p=4$"))

    // Verify round-trip
    ok, err := VerifyPassword("test-password", hash)
    require.NoError(t, err)
    assert.True(t, ok)

    // Verify wrong password returns false, not error
    ok, err = VerifyPassword("wrong-password", hash)
    require.NoError(t, err)
    assert.False(t, ok)

    // Verify constant-time (timing test — approximate)
    // This is a smoke test; real timing analysis requires specialized tools
    start := time.Now()
    VerifyPassword("wrong", hash)
    wrongDuration := time.Since(start)

    start = time.Now()
    VerifyPassword("test-password", hash)
    correctDuration := time.Since(start)

    // Both should take approximately the same time (within 10%)
    ratio := float64(wrongDuration) / float64(correctDuration)
    assert.InDelta(t, 1.0, ratio, 0.1, "timing difference suggests non-constant-time comparison")
}

func TestRandomEntropy(t *testing.T) {
    // Verify 256 bits of entropy in generated tokens
    tokens := make(map[string]bool)
    for i := 0; i < 10000; i++ {
        token := GenerateRandomBase62(32)
        assert.False(t, tokens[token], "collision detected — entropy insufficient")
        tokens[token] = true
    }
}
```

### 4. Migrate existing hash usage to `internal/crypto`

Update `services/auth/internal/service/` to use `internal/crypto.HashToken`
and `internal/crypto.VerifyToken` instead of any inline Argon2id calls.

### 5. Document parameter upgrade procedure

In ADR-0008, document the procedure for upgrading Argon2id parameters:

1. Deploy new code that can verify both old and new parameter sets
2. On successful login with old parameters: rehash with new parameters
3. After 90 days: remove old parameter support
4. Update ADR with new parameters
