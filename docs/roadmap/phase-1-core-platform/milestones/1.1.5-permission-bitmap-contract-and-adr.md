# Milestone 1.1.5: Permission Bitmap Contract and ADR

**Status:** Planned
**Goal:** [1.1 Persistence and auth data plane](../goals.md)
**Phase:** 1 — Core Platform

## Why This Milestone Exists

The 64-bit permission bitmap is referenced in:
- `ARCHITECTURE.md §5` (bit layout defined)
- `DATABASE_SCHEMA.md` (tokens.permissions BIGINT)
- `AGENTS.md §5.3` (every endpoint must declare required permissions)
- `SECURITY.md §4.3` (authorization model)
- `decisions.md` (pending: "which bits required for proxy chat completion in Phase 2")

But there is no ADR, no Go constants file, no test that verifies the bitmap
semantics, and no documentation of which bits are required for which operations.

Without this milestone, every service that checks permissions will invent its own
constants, creating drift. A permission check bug in a security-critical path
(proxy auth, memory access) is a tenant isolation failure.

This milestone creates the single source of truth for the permission bitmap.

### 1. Write ADR-0007: Permission Bitmap Layout

Document the full 64-bit layout with rationale for each group:

```text
Bit Position | Group              | Permissions
-------------|--------------------|-----------------------------------------
0            | Memory             | MEMORY_READ
1            | Memory             | MEMORY_WRITE
2            | Memory             | MEMORY_DELETE
3            | Memory             | MEMORY_BULK_EXPORT
4-7          | Memory             | Reserved
8            | Directive          | DIRECTIVE_READ
9            | Directive          | DIRECTIVE_WRITE
10           | Directive          | DIRECTIVE_PROMOTE (requires MFA)
11           | Directive          | DIRECTIVE_REVOKE (requires MFA)
12-15        | Directive          | Reserved
16           | Session            | SESSION_CREATE
17           | Session            | SESSION_READ
18           | Session            | SESSION_TERMINATE
19-23        | Session            | Reserved
24           | Trace              | TRACE_READ
25           | Trace              | TRACE_EXPORT
26-31        | Trace              | Reserved
32           | Admin              | USER_MANAGE
33           | Admin              | BILLING_READ
34           | Admin              | BILLING_MANAGE
35           | Admin              | ORG_SETTINGS_WRITE
36           | Admin              | TOKEN_CREATE
37           | Admin              | TOKEN_REVOKE
38-39        | Admin              | Reserved
40           | Marketplace        | MARKETPLACE_PUBLISH
41           | Marketplace        | MARKETPLACE_INSTALL
42-47        | Marketplace        | Reserved
48           | Federation         | FEDERATION_SHARE
49-55        | Federation         | Reserved
56-63        | Reserved           | Do not use
```

**Phase 2 minimum set** (resolves pending decision in `decisions.md`):
For proxy chat completion, a token must have: `MEMORY_READ | SESSION_CREATE | SESSION_READ`

Document predefined permission sets:
- `PERMISSIONS_AGENT_DEFAULT`: `MEMORY_READ | MEMORY_WRITE | SESSION_CREATE | SESSION_READ | TRACE_READ`
- `PERMISSIONS_ADMIN`: all bits in groups 0-39
- `PERMISSIONS_READONLY`: `MEMORY_READ | SESSION_READ | TRACE_READ | DIRECTIVE_READ`

### 2. Create `packages/permissions/permissions.go`

```go
// Package permissions defines the IBEX Harness permission bitmap.
// This is the single source of truth for all permission constants.
// See docs/adr/ADR-0007-permission-bitmap.md for the full specification.
package permissions

const (
    // Memory operations (bits 0-7)
    MemoryRead       int64 = 1 << 0
    MemoryWrite      int64 = 1 << 1
    MemoryDelete     int64 = 1 << 2
    MemoryBulkExport int64 = 1 << 3

    // Directive operations (bits 8-15)
    DirectiveRead    int64 = 1 << 8
    DirectiveWrite   int64 = 1 << 9
    DirectivePromote int64 = 1 << 10  // requires MFA
    DirectiveRevoke  int64 = 1 << 11  // requires MFA

    // Session operations (bits 16-23)
    SessionCreate    int64 = 1 << 16
    SessionRead      int64 = 1 << 17
    SessionTerminate int64 = 1 << 18

    // Trace operations (bits 24-31)
    TraceRead   int64 = 1 << 24
    TraceExport int64 = 1 << 25

    // Admin operations (bits 32-39)
    UserManage        int64 = 1 << 32
    BillingRead       int64 = 1 << 33
    BillingManage     int64 = 1 << 34
    OrgSettingsWrite  int64 = 1 << 35
    TokenCreate       int64 = 1 << 36
    TokenRevoke       int64 = 1 << 37

    // Marketplace operations (bits 40-47)
    MarketplacePublish int64 = 1 << 40
    MarketplaceInstall int64 = 1 << 41

    // Federation operations (bits 48-55)
    FederationShare int64 = 1 << 48
)

// Predefined permission sets
const (
    // AgentDefault is the minimum permission set for a production agent.
    AgentDefault = MemoryRead | MemoryWrite | SessionCreate | SessionRead | TraceRead

    // ProxyChatCompletion is the minimum required for proxy chat completion (Phase 2).
    ProxyChatCompletion = MemoryRead | SessionCreate | SessionRead

    // ReadOnly grants read access to all non-admin resources.
    ReadOnly = MemoryRead | DirectiveRead | SessionRead | TraceRead

    // Admin grants all non-federation, non-marketplace permissions.
    Admin = AgentDefault | DirectiveRead | DirectiveWrite | DirectivePromote |
            DirectiveRevoke | SessionTerminate | TraceExport |
            UserManage | BillingRead | BillingManage | OrgSettingsWrite |
            TokenCreate | TokenRevoke
)

// Has returns true if the bitmap includes all of the required permissions.
func Has(bitmap, required int64) bool {
    return bitmap&required == required
}

// HasAny returns true if the bitmap includes at least one of the given permissions.
func HasAny(bitmap int64, permissions ...int64) bool {
    for _, p := range permissions {
        if bitmap&p != 0 {
            return true
        }
    }
    return false
}

// RequiresMFA returns true if the permission requires MFA verification.
func RequiresMFA(permission int64) bool {
    return permission&(DirectivePromote|DirectiveRevoke) != 0
}
```

### 4. Write comprehensive unit tests

```go
func TestPermissionBitmap(t *testing.T) {
    // Test Has()
    assert.True(t, Has(AgentDefault, MemoryRead))
    assert.False(t, Has(AgentDefault, BillingManage))

    // Test no overlap between groups
    assert.Equal(t, int64(0), MemoryRead & DirectiveRead)

    // Test predefined sets are subsets of Admin
    assert.Equal(t, AgentDefault, AgentDefault & Admin)

    // Test ProxyChatCompletion is subset of AgentDefault
    assert.Equal(t, ProxyChatCompletion, ProxyChatCompletion & AgentDefault)

    // Test MFA requirements
    assert.True(t, RequiresMFA(DirectivePromote))
    assert.False(t, RequiresMFA(MemoryRead))

    // Test no reserved bits are set in predefined sets
    const reservedMask = int64(0xFF00000000000000) // bits 56-63
    assert.Equal(t, int64(0), Admin & reservedMask)
}
```
