Personal Access Tokens (PATs) are the Phase 1 client credential for IBEX Harness. Each token is scoped to an organization, carries a permission bitmap, and is verified by Argon2id hash lookup — only the hash is stored; the plaintext is shown **once** at creation ([ADR-0007](/docs/adr/0007-auth-token-validation), [ADR-0010](/docs/adr/0010-cryptography-policy)).

<Callout type="danger" title="Secret handling">
  Never log, commit, or paste PATs into tickets. Store in environment variables or a secrets manager. Log only an 8-character prefix in application code — see [Secrets and keys](/docs/security/secrets-and-keys).
</Callout>

## PAT wire format

```
ibex_pat_<token_uuid>_<secret>
```

Example dev token (from `make db-seed`):

```
ibex_pat_00000000-0000-0000-0000-000000000004_LOCALDEVELOPMENTONLY
```

The auth service parses the prefix, looks up `token_uuid` in `ibex_core.tokens`, and verifies the secret with Argon2id. Revoked tokens fail immediately regardless of hash match.

## Permission bitmap

Tokens encode capabilities as a 64-bit bitmap ([ADR-0009](/docs/adr/0009-permission-bitmap)). Common bits for proxy integrators:

| Bit | Name | Needed for |
| --- | --- | --- |
| — | `ProxyChatCompletion` | `POST /v1/chat/completions` |
| — | `TokenCreate` | `CreateToken` gRPC |
| — | `TokenRevoke` | `RevokeToken` gRPC |

`ProxyChatCompletion` is a composite of memory/session read bits — sufficient for Phase 1 chat probes.

<Steps>
  <Step title="Migrate and seed">
    `make db-migrate && make db-seed` — idempotent; refuses production DSNs.
  </Step>
  <Step title="Export env vars">
    Seed output sets `IBEX_DEV_TOKEN`, `IBEX_DEV_AGENT_ID`, `IBEX_DEV_ORG_ID`.
  </Step>
  <Step title="Smoke test">
    `make dev-smoke` — validates probes and chat stub with seeded credentials.
  </Step>
</Steps>

Fixed seed IDs: org `00000000-0000-0000-0000-000000000001`, agent `00000000-0000-0000-0000-000000000003`.

## Create via gRPC

`CreateToken` requires an existing admin bearer with `TokenCreate` permission.

<CodeTabs>
  <CodeTab label="grpcurl">
```bash
grpcurl -plaintext \
  -H "authorization: Bearer ibex_pat_<admin-uuid>_<secret>" \
  -d '{"org_id":"<org-uuid>","name":"dev-pat","type":1,"permissions":23}' \
  localhost:9091 ibex.auth.v1.AuthService/CreateToken
```
  </CodeTab>
  <CodeTab label="Validate (no auth)">
```bash
grpcurl -plaintext \
  -d '{"access_token":"ibex_pat_<uuid>_<secret>"}' \
  localhost:9091 ibex.auth.v1.AuthService/ValidateToken
```
  </CodeTab>
</CodeTabs>

The response field `plaintext` is the only chance to copy the secret. Persist it immediately.

## Revoke a token

Immediate invalidation — no grace period:

```bash
grpcurl -plaintext \
  -H "authorization: Bearer ibex_pat_<admin-uuid>_<secret>" \
  -d '{"org_id":"<org-uuid>","token_id":"<token-uuid>"}' \
  localhost:9091 ibex.auth.v1.AuthService/RevokeToken
```

Revoked tokens return `401` on the next proxy request.

## Rotation procedure

<ProcessSteps
  steps={[
    {
      title: 'Issue replacement',
      description: 'CreateToken with the same org and desired permissions.',
    },
    {
      title: 'Deploy new secret',
      description: 'Update client env vars or secrets manager entry.',
    },
    {
      title: 'Verify traffic',
      description: 'Probe /v1/internal/auth-probe with the new PAT.',
    },
    {
      title: 'Revoke old token',
      description: 'RevokeToken on the previous token_id — instant cutover.',
    },
  ]}
/>

Never rotate by deleting rows in Postgres — use the gRPC API so audit and revocation caches stay consistent.

## Operator checklist

<Callout type="tip" title="Production readiness">
  Before promoting a PAT beyond dev: confirm permissions are minimal, expiry policy is documented, and rotation runbook is assigned.
</Callout>

- [ ] Permissions follow least privilege (`ProxyChatCompletion` only for LLM clients)
- [ ] Token name identifies the owning service or agent
- [ ] Plaintext stored in vault, not `.env` committed to git
- [ ] Revocation tested in staging
